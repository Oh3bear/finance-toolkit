## 一、内部往来核对

### 1.1 用途说明

用于**利润中心之间往来款项的自动核对**。例如：

- A利润中心账上记录"应收B中心 100万"
- B利润中心账上记录"应付A中心 100万"
- 系统自动匹配这两笔记录，标记为"已对符"

**核心能力**：支持 **M:N 匹配**，即多笔小金额可以合并后与一笔大金额匹配，解决实际业务中一笔总账对应多笔明细的情况。

### 1.2 数据来源

需要准备两份Excel文件：

| 文件 | 内容 | 必填列 |
|------|------|--------|
| **明细账** | 各利润中心的往来明细记录 | 公司名称、科目名称、对方单位、金额（借/贷） |
| **映射表** | 利润中心名称与代码的对照关系 | 代码、名称（用于识别同一单位的不同叫法） |

> 映射表为可选，若利润中心名称规范统一，可直接在页面手动维护映射关系。

### 1.3 操作步骤

**第一步：数据导入**

1. 点击左侧菜单"内部往来核对"
2. 在"导入明细账"区域上传往来明细Excel
3. 在"导入映射表"区域上传映射对照Excel（可选）
4. 点击右下角"下一步：维护映射"

**第二步：映射维护**

1. 系统会自动识别双方账簿中的公司/利润中心名称
2. 检查左侧（账簿A）与右侧（账簿B）的对应关系是否正确
3. 如有未匹配的单位，手动点击下拉框进行配对
4. 确认无误后，点击"下一步：执行核对"

**第三步：执行核对**

1. 核对参数一般保持默认即可
   - 匹配模式：严格（金额完全一致）或 容差（允许小额差异）
   - 容差金额：如设置为 0.01，则差异在 1 分钱以内视为已对符
2. 点击"开始核对"
3. 等待进度完成，系统会自动跳转至结果页

**第四步：核对结果**

结果页面展示四项统计：

| 状态 | 含义 |
|------|------|
| **已对符** | 双方记录金额一致，已自动匹配 |
| **仅账簿A有** | 只在左侧账簿出现，右侧无对应记录 |
| **仅账簿B有** | 只在右侧账簿出现，左侧无对应记录 |
| **金额不符** | 双方都有记录，但金额不一致 |

- 点击"导出"可下载完整核对结果Excel
- 表格支持按状态筛选、按金额/公司名称排序
- 可逐行查看匹配详情

## 二、银企对账

### 2.1 用途说明

用于**银行流水与企业银行明细账的逐笔自动核对**。解决财务人员每月需要人工比对银行对账单与企业账务记录的工作。

**核心能力**：
- 支持导入 **PDF银行对账单** 或 **Excel银行流水**
- 智能匹配算法，自动处理日期偏移、多笔合并等复杂场景
- 自动计算余额差异，定位未达账项

### 2.2 数据来源

| 文件 | 格式 | 内容 |
|------|------|------|
| **银行流水** | PDF（银行对账单）或 Excel | 银行提供的对账单/流水明细 |
| **企业账** | Excel | 企业财务系统中的银行明细账 |

### 2.3 操作步骤

**第一步：导入银行流水**

1. 点击左侧菜单"银企对账"
2. 在"银行流水"区域，上传银行对账单PDF或Excel
3. 系统自动解析，识别账户号码、交易日期、金额、摘要等字段
4. 核对解析结果，确认列映射正确
5. 点击"下一步：导入企业账"

**第二步：导入企业账**

1. 在"企业银行明细账"区域，上传企业账务Excel
2. 系统同样会自动解析并识别字段
3. 确认列映射无误后，点击"下一步：执行核对"

**第三步：执行核对**

1. 设置核对参数（通常保持默认）：
   - **日期容差**：允许交易日期相差的天数（如 3 天）
   - **金额容差**：允许金额差异（如 0.01 元）
2. 点击"开始核对"
3. 系统会自动执行多阶段匹配：
   - 快速通道：总金额完全一致则整批匹配
   - 冲销处理：自动识别同方向正负对冲记录
   - 1:1逐笔匹配：按金额+日期精确匹配
   - M:N组合匹配：多笔小金额合并匹配一笔大金额
   - 差额填补：近匹配记录通过补充条目完成匹配
4. 等待进度完成，跳转结果页

**第四步：核对结果**

结果按**账户**分组展示：

| 状态 | 含义 |
|------|------|
| **已对符** | 银行流水与企业账记录匹配成功 |
| **未对符（银行）** | 银行流水中有，企业账中无对应记录 |
| **未对符（企业）** | 企业账中有，银行流水中无对应记录 |

**导出功能**：
- 点击"导出Excel"，生成包含以下Sheet的文件：
  - **账户列表**：汇总各账户的对符状态，点击"查看明细"可跳转
  - **各账户明细**：每个账户的匹配/未匹配记录详情
- 导出文件中的"查看明细"列为超链接，点击可直接跳转到对应Sheet

## 三、PDF处理

### 3.1 PDF合并

#### 用途说明

将**多个PDF文件合并为一个PDF**，常用于：
- 将分散的月度报表合并为年度汇总
- 将多个合同附件合并为一个文件归档
- 按分组合并（如按项目、按月份）

支持添加**分隔页**和**页码**，方便打印和查阅。

#### 操作步骤

1. 点击左侧菜单"PDF合并"
2. 点击上传区域，选择一个或多个PDF文件
3. 文件上传后，可在列表中拖拽调整顺序
4. 配置选项：
   - **添加分隔页**：在每个文件前插入带有文件名的分隔页
   - **添加页码**：在页面底部居中添加页码
5. 点击"合并并下载"
6. 浏览器自动下载合并后的PDF文件

---

### 3.2 PDF 2合1

#### 用途说明

将PDF的**每2页合并为1页**（并排显示），常用于：
- 将A4打印稿转为手册式排版，节省纸张
- 缩小文件页数，方便传阅
- 保留原始文件名，便于归档识别

#### 操作步骤

1. 点击左侧菜单"PDF 2合1"
2. 点击上传区域，选择一个PDF文件
3. 系统自动处理，将每2页横向并排合并为1页
4. 处理完成后，自动下载新PDF
5. 新文件名格式：`原文件名_2合1.pdf`

## 常见问题

**Q1：数据是否会泄露？**
> 不会。本工具为纯前端应用，所有数据仅在您的浏览器中处理，不会上传到任何服务器。您可以断网使用。

**Q2：支持哪些Excel格式？**
> 支持 `.xlsx` 和 `.xls` 格式。建议优先使用 `.xlsx`。

**Q3：银行对账单PDF解析失败怎么办？**
> 请确保上传的是标准文本型PDF（而非扫描件图片）。如果解析结果不准确，可在"数据导入"步骤手动调整列映射。

**Q4：核对结果中"金额不符"如何处理？**
> 建议先检查双方账务是否包含手续费、利息等差异项。如有容差设置，可适当放宽容差范围后重新核对。
